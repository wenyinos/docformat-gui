# ============================================
# 本单位术语 / 背景表（ai_glossary.txt）
# 作用：告诉 AI 哪些写法是本单位已知正确的，避免它把内部简称、
#       惯用说法当成错误来纠（减少误报）。
#
# 重要：本文件内容会随每次 AI 检查一起发送到 AI 服务商的服务器。
#       因此只写"东西怎么称呼、单位是什么情况"，不要写"人是什么样"。
#       可以写：简称全称、惯用公文用语、上级单位、所在区划、单位层级、
#             需要校对的部门名称、人员姓名和职务称谓。
#       不要写：员工性格、个人评价、内部人事安排、涉密信息。
#
# 用法：把本文件复制改名为 ai_glossary.txt，按下面格式填本单位内容。
#       # 开头的行是注释，不会被发送；【方括号】只是分类标题，可改可删；
#       正文一行一条大白话即可，不需要任何特殊符号或嵌套。
#       默认按本单位发文理解；如果部门、人名有常见误写，可以写明正确写法。
# ============================================

【单位简称 / 全称】
示例：xx集团有限公司，简称"xx集团"
示例：xx管理委员会，简称"xx管委"

【上级单位 / 层级 / 区划】
示例：主管单位为 xx 市国资委
示例：本单位为 xx 区直属单位，不是最高级单位

【固定 / 惯用公文用语】
示例：通知类结尾统一用"特此通知"
示例：请示类结尾用"妥否，请批示"

【部门与人员称谓】（只写名称和称呼，不写对人的评价）
示例：投资部、综合办公室、财务部
示例：正确部门名称为"综合办公室"，不要写成"综合办公司"
示例：张三，职务称谓为"项目负责人"
示例：李四，常用称谓为"李四同志"
